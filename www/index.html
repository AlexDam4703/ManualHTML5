Estructura básica de un documento HTML

Al ser un lenguaje de marcas, la sintaxis de HTML funciona de una manera similar a XHTML, es decir, se compone de etiquetas y tiene una estructura jerarquizada. Sin embargo, hay unas cuantas diferencias a tener en cuenta:

  • Los atributos en XHTML deben tener un valor pero en HTML no es obligatorio darles uno.
  • Los valores de los atributos en XHTML deben ir siempre entrecomillados, mientras que en HTML no es necesario al menos que están separados por espacios en blanco.
  • Las etiquetas deben cerrarse siempre en XHTML y deben escribirse en minúsculas. Sin embargo, el cierre en HTML no es obligatorio y la etiqueta se puede escribir tanto en mayúsculas como en minúsculas.

De esta manera, la estructura del código es más accesible y flexible. Al hablar de HTML, hay tres etiquetas que son obligatorias para su interpretación, las cuales siguen un orden:

  1. <html>: Es el nombre que se le da a la etiqueta raíz. Normalmente suele tener un atributo "lang" para especificar el idioma. Esta etiqueta actúa como elemento padre de las etiquetas <head> y <body>
  2. <head>: Delimita el código que será utilizado para la cabecera de la página. Aquí se escribe el título de ésta usando la etiqueta <title> y se codifica el texto con la etiqueta <meta>
  3. <body>: Bajo esta etiqueta se escribe todo el contenido visible de la página. Para que un texto sea visible en el navegador sólo hace falta escribirlo aquí.

Con estos conocimientos cualquiera puede crear una página web básica, aunque se requiere y se pide el uso de más etiquetas para crear webs más completas y atractivas a la vista. Por otro lado, la complejidad del código puede dificultar el trabajo, así que los programadores utilizan comentarios en el mismo código para no perderse. El comentario tiene la siguiente forma: <!-- TEXTO A ESCRIBIR>. El comentario no será leído por el navegador.

Formato básico de texto

Formato de texto

Las etiquetas <strong> </strong> o <b> </b> se utilizan para remarcar en negrita lo que haya dentro.

Las etiquetas <em> </em> o <i> </i>se utilizan para poner en cursiva lo que hay dentro.

La etiqueta <ins> </ins> se utiliza para subrayar un texto.

La etiqueta <del> </del> se utiliza para tachar un texto.

La etiqueta <sub> </sub> se utiliza para que el texto se encuentre como un subíndice, como podemos observar en los números de esta fórmula: Sb2O3.

La etiqueta <sup> </sup> se utiliza para poner el texto como un superíndice, como el 3 en esta expresión: 103.

La etiqueta <mark> </mark> sirve para remarcar un texto con el mismo efecto que tendría un rotulador fluorescente.

La etiqueta <small> </small> se utiliza para hacer más pequeño el tamaño del texto.

Títulos y párrafos

Párrafos

Para definir párrafos, se utiliza la etiqueta <p> </p>.

La etiqueta <br/> se utiliza para dar un salto de línea, como el que se puede observar debajo de esta frase.


Nota: en HTML, los saltos de línea en el código escritos con la tecla Enter no se verán finalmente en la página en el navegador. Es por ello que hay que utilizar etiquetas como <p> </p> o <br/>.

La etiqueta <hr/> separa el texto con una línea horizontal.

Títulos

Para definir el tamaño de los títulos, se utilizan las etiquetas <h1> </h1>, <h2> </h2>... sucesivamente hasta <h6> </h6>. Cuanto menor sea el número que acompaña a la letra h de la etiqueta, mayor tamaño tendrá la letra.

Etiquetas semánticas

Las etiquetas semánticas se encargan de englobar los contenidos de una página web, ayundando a definir la estructura del documento, son semánticas ya que no indican la forma con la cual se debe mostrar, sino el significado que toman dentro del contexto de la página.

Existen varios tipos de etiquetas semánticas:

La etiqueta <header> </header> especifica una cabecera para una página o sección, la cual debería contener un elemento introductorio. Puede haber varios <header> en la misma página

<header>

     <h1>Esta es la cabecera de mi página</h1>

</header>

Dentro de la etiqueta <nav> </nav> se incluirán todos los enlaces necesarios para que el usuario pueda moverse cómodamente por nuestro sitio web

<nav>

     <p>Aquí se pueden insertar todos los enlaces para moverse por las distintas páginas de nuestro sitio web </p>

</nav>

La etiqueta <section> </section> se encarga de definir las diferentes secciones de nuestra página.

<section>

     <h1>Esta es una sección de nuestra página</h1>

          <p>Aquí se incluirá todo el contenido de esta sección</p>

</section>

La etiqueta <article> </article> nos ayuda a definir un contenido específico dentro de nuestra web que tiene sentido por si solo. Por ejemplo se podría usar para el post de un foro o blog o para un artículo de un periódico

<article>

     <h2>Titular</h2>

          <p>Contenido referente al titular</p>

</article>

La etiqueta <aside> </aside> es utilizada para definir un contenido que se considera separado del contenido principal, pero está relacionado con este.

<aside>

     <p>Aquí podríamos introducir por ejemplo enlaces externos.</p>

</aside>

La etiqueta <footer> </footer> se encarga de especificar el pie de página de una página web o sección

<footer>

     <p>Aquí se puede insertar por ejemplo el autor y su email de contacto</p>

</footer>

Además de las etiquetas anteriores también contamos con otras etiquetas como<figure> </figure>,para contenido multimedia dentro de la cual podemos incluir <figcaption> </figcaption> para dar información sobre el elemento figure que puede ser por ejemplo una imagen, <details> </details> para definir detalles adicionales que el usuario puede ver o no, <main> </main> para especificar el contenido principal de un documento, <date> </date> para definir la fecha/hora, <mark> </mark> para definir un contenido marcado, <summary> </summary> para definir un encabezado visible para un elemento <details> </details>

Recordemos de nuevo que estas etiquetas por si mismas no modifican nada visualmente, por lo que si se escriben los códigos anteriores solo se podrá apreciar un <p>, un <h1>, etc.

Listas

Listas ordenadas.

Las listas desordenadas o listas con numéricas, se forman con las etiquetas <ul> y cada uno de los elementos de la lista tendran la etiqueta <li> quedando de esta forma:

  1. Don Quijote
  2. Los girasoles ciegos
  3. Los pilares de la Tierra

Siendo su código: 

            <ol>
                 <li>Don Quijote<li>
                 <li>Los girasoles ciegos<li>
                 <li>Los pilares de la Tierra<li>
             <ol>

Para dar formato a la lista podemos usar un atributo quedado de la siguiente manera < ol type="">

Usaremos los atributos "Square" para usar cuadrados, "disc" para el uso de puntos, "circle" para circulos, "1" para usar numeracion decimal "a|A", para el uso de letras.

Las listas desordenadas o listas de viñetas se forman con la etiqueta <ul> seguida de las etiquetas <li> para cada uno de los elementos que la forman quedando de la siguiente forma:

Tablas

En HTML se pueden crear tablas utilizando la etiqueta <table>. También se le debe indicar al navegador el número de filas y columnas que la componen, además del contenido de las celdas. Para ello se utilizan cuatro etiquetas:

  • <caption> sirve para poner un título a la tabla. Esta etiqueta debe ir justo después de <table> para que sea interpretada
  • <tr> sirve para definir una fila
  • <th> sirve para definir una cabecera, que sirve como la primera celda para una columna. No son necesarias para estructurar una tabla pero son recomendables para que la tabla se entienda
  • <td> sirve para definir una celda. A diferencia de los editores de texto, los datos de las celdas de una misma fila sólo se codifican de izquierda a derecha

Para crear una tabla, primero se determinan las filas que se necesiten, después se determinan las cabeceras en la primera fila y por último las celdas en las siguientes. Haz click aquí para ver un ejemplo de una tabla básica

Las tablas pueden modificarse de forma más específica utilizando unas propiedades o atributos reservados para éstas. Tanto los atributos como las propiedades pueden codificarse para toda la tabla, una o varias filas sólo o una o varias celdas

Un texto es una composición de signos codificados en un sistema de escritura que forma una unidad de sentido.

También es una composición de caracteres imprimibles (con grafema) generados por un algoritmo de cifrado que, aunque no tienen sentido para cualquier persona, sí puede ser descifrado por su destinatario original. En otras palabras, un texto es un entramado de signos con una intención comunicativa que adquiere sentido en determinado contexto.

Las ideas esenciales que comunica un texto están contenidas en lo que se suele denominar «macroproposiciones», unidades estructurales de nivel superior o global, que otorgan coherencia al texto constituyendo su hilo central, el esqueleto estructural que cohesiona elementos lingüísticos formales de alto nivel, como los títulos y subtítulos, la secuencia de párrafos, etc. En contraste, las «microproposiciones» son los elementos coayudantes de la cohesión de un texto, pero a nivel más particular o local. El nivel microestructural o local está asociado con el concepto de cohesión. Se refiere a uno de los fenómenos propios de la coherencia, el de las relaciones particulares y locales que se dan entre elementos lingüísticos, tanto los que remiten unos a otros como los que tienen la función de conectar y organizar. También es un conjunto de oraciones agrupadas en párrafos que habla de un tema determinado.

URLs

Una URL es la direción donde se encuentra una determinada información localizada en internet. Cada URL tiene que ser única convirtiendose en un identificador como el "DNI" de una persona.

    Una URL se forma usando:
  • Protocolo
    • http:// (recursos de la web)
    • https:// (recursos que tienen contenidos seguros)
    • ftp:// (datos contenidos en un servidor)
    • ftps:// (datos contenidos en un servidor seguro)
    • mailto: (Dirección de correo electrónico)
    • file:/// (Datos que se encuentran dentro del mismo ordenador)
  • Servidor
    • Nombre del servidor donde estan almacenados los datos, por ejemplo : "www.iesviergendelcarmen.com".
  • Puerto
    • Puerto al que se debe conectar para que el servidor obtenga los datos, este por norma general no se especifíca sindo por defecto el 80.
  • Ruta
    • Indica porque directorios hay que pasar para llegar a los datos que queremos conseguir. Por ejemplo: /home/foto/fotoHTML accede a fotoHTML que esta dentro del directorio fotos y esta a su vez dentro de home.
  • Cadena de búsqueda
    • Sirve para pasar parámetros a aquellas paginas que lo permitan

Por otro lado, tenemos que tener encuenta la codificación de las rutas URL que deben estar siempre codificados en ASCII.

  10. prev Si nuestra página pertenece a una serie ordenada, este enlace nos llevará al enlance anterior
  11. tag Permite el uso de etiquetas (tags)

Gradientes

Una gradiente es un relleno que va cambiando progresivamente de color. Por lo tanto, se debe especificar un relleno para el objeto antes de dibujarlo: Para crear una gradiente se necesita definir una gradiente y luego indicar que debe ser usada. Se pueden utilizar dos tipos de gradientes:

  • Gradiente linear: es una gradiente que se extiende en base a un eje. Para crearla se utiliza el comando "var NOMBRE DE LA GRADIENTE=ctx.createLinearGradient(A, B, C, D, ...);" Los valores A y B son las coordenadas X e Y, respectivamente, de la esquina superior izquierda de la gradiente, mientras que C, D, E, F... representan las coordenadas X donde termina un color y pasa al siguiente, según el orden de aparición
  • Gradiente radial: es una gradiente que tiene forma circular y parte de un centro. Para crearla se utiliza el comando "var NOMBRE DE LA GRADIENTE=ctx.createRadialGradient(A, B, C, D, ..., E, F);". Los valores A Y B son las coordenadas X e Y, respectivamente, del centro de la gradiente. C, D, ... son los radios del centro de cada color, mientras que E y F son las coordenadas de la gradiente en sí, tomando como punto de referencia el centro

Además, se le debe indicar al navegador el orden de aparición de los colores. Para ello se añade la línea "NOMBRE DE LA GRADIENTE.addColorStop(0-1,"COLOR EN INGLÉS");" tantas veces como colores haya. A cada color se le asigna un número del 0 al 1 y aparecerón en orden ascendente

Con estos comandos se define una gradiente cualquiera. Para que la gradiente aparezca en la página se le debe asignar un objeto y un área dentro de éste. Para ello se usan los siguientes comandos

  • "ctx.fillStyle = NOMBRE DE LA GRADIENTE;" se usa para indicar cuál gradiente va a dibujarse en el objeto
  • "ctx.fill();" se usa para indicar en qué área del objeto aparecerá la gradiente. Las coordenadas de la gradiente dependen del tipo de objeto que colorea

WCAG 1.0 y 2.0

La Iniciativa de Accesibilidad a la Web del W3C (WAI) es un grupo de trabajo permanente del W3C dedicado a promover soluciones de accesibilidad en la web para personas discapacitadas.

La WAI para facilitar la accesibilidad web ha propuesto una serie de pautas a seguir, las WCAG 1.0, publicadas en 1999 y las WCAG 2.0, publicadas en 2008.

Las WCAG 2.0 se basan fundamentalmente en 4 principios.

Perceptible

La información se debe presentar al usuario de forma que sea perceptible para ellos, por ejemplo dando subtítulos y otras alternativas para los contenidos multimedia, creando un contenido que se pueda presentar de diferentes formas, incluyendo las tecnologías asistidas sin perder su significado, dando alternativas textuales para contenido no textual, como textos ampliados, voz, braille, símbolos, etc, y facilitarle a los usuarios el oir y ver el contenido.

Operable

Los componentes de la interfaz y navegación deben ser operables, se debe hacer que toda funcionalidad sea accesible mediante el teclado, se debe dar el tiempo suficiente para que los usuarios lean y usen el contenido, no se debe diseñar contenido que podría provocar ataques, espasmos o convulsiones y se debe proporcionar ayuda a los usuarios a la hora de navegar y encontrar contenido.

Comprensible

La información y el manejo de la interfaz debe ser comprensible. El texto debe ser legible y comprensible, las páginas deben aparecer y operar de manera predecible y se debe ayudar a los usuarios a evitar y corregir los errores.

Robusto

Se debe maximizar la compatibilidad con los navegadores y las herramientas de usuario actuales y futuras, incluyendo las ayudas ténicas.

Accesibilidad web

¿Qué es la accesibilidad web?

Cuando nos referimos a la accesibilidad web hablamos de facilitar el acceso a la web independientemente del tipo del hadware, software, infraestructura de red, idioma, cultura etc de los usuarios.

Con esta idea es como nace la Iniciativa de Accesibilidad web, conocida como WAI. Cuyo objetivo es el de facilitar el acceso a las persona con discapacidad.

¿Para qué sirve?

La idea principal es la de hacer que toda página web sea accesible para todos los usuarios independientemente de la circustancias y de los dispositivos que usen para acceder a la información.

¿Cómo funciona?

Para que el contenido de una página web sea accesible se han desarrollado una serie de pautas denominadas Pautas de accesibilidd al contenido web que consisten en 12 pautas y 4 principios que proporcionan soluciones de diseño. Además las pautas constan de una serie de puntos de verificación que ayudan a detectar posibles errores

Cada punto de verificación esta asiganado a uno de los tres niveles de prioridad establecidos por pautas:

  1. Un desarrollador de contenidos de páginas Web tiene que satisfacer este punto de verificación. De otra forma, uno o más grupos de usuarios encontrarán imposible acceder a la información del documento. Satisfacer este punto de verificación es un requerimiento básico para que algunos grupos puedan usar los documentos Web.
  2. Un desarrollador de contenidos de páginas Web debería satisfacer este punto de verificación. De otra forma, uno o más grupos encontrarán dificultades en el acceso a la información del documento. Satisfacer este punto de verificación eliminará importantes barreras de acceso a los documentos Web.
  3. Un desarrollador de contenidos de páginas Web puede satisfacer este punto de verificación. De otra forma, uno o más grupos de usuarios encontrarán alguna dificultad para acceder a la información del documento. Satisfacer este punto de verificación mejorará la accesibilidad de los documentos Web.

En función a estos puntos de verificación se establecen los niveles de conformidad:

  1. Nivel de conformidad "A": Todos los puntos de verificación de prioridad 1 se satisfacen
  2. Nivel de conformidad "AA": Todos los puntos de verificación de prioridad 1 y 2 se satisfacen
  3. Nivel de conformidad "AAA": Todos los puntos de verificación de prioridad 1, 2 y 3 se satisfacen

Las pautas describen cómo hacer páginas Web accesibles sin sacrificar el diseño, ofreciendo esa flexibilidad que es necesaria para que la información sea accesible bajo diferentes situaciones y proporcionando métodos que permiten su transformación en páginas útiles e inteligibles.

Pautas de accesibilidad al contenido web

Principio 1: Perceptible
Toda la información de la interfaz deber de ser presentados de modo que sean visible para todos los usuarios. Este principio se divide en 4 pautas:
  1. Alternativas textuales Proporcionar alternativas textuales para todo contenido no textual. Por ejemplo imágenes, representaciones gráficas etc. Cabe destacar que la información resultante debe de transmitir la misma información.
  2. Medios dependiente del tiempo Proporciona alternativas para los medios que dependan del tiempo. Por ejemlo todo video debe de ir acompañado de subtitulos
  3. Adaptable Crear contenido que se pueda representar de varias formas para así adapatarse a la dificiltad
  4. Distinguible Se debe facilitar a los usuarios ver o escuchar el contenido haciendo que estos contrasten con un fondo. Por ejemplo un texto debe contrastar con el fondo para que así sea mas legible
Principio 2: Operable
La interfaz NO debe contener una interacción que el usuario no pueda hacer. Este principio se divide en cuantro pautas:
  1. accesible por teclado La interfaz es totalmente accesible por teclado
  2. Tiempo suficiente Se debe proporcionar tiempo suficiente para leer y usar el contendio de la página
  3. Convulsiones La pagina puede crear contenido que pueda crear convulsiones.
  4. Navegación La página debe contar con los medios para localizar de forma rapida y accesible la información de la misma
Principio 3: Comprensible
El contenido de la página no debe ir mas allá de la compresión del usuario
  1. Legible Radactar los textos de forma que sean legibles y comprensibles para el usuario
  2. Predecible Las páginas web deben de comportarse de forma coherente de cara al usuario
  3. Entrada de información asistida La página debe contar con los medios para ayudar al usuario en caso de una mala entrada de datos
Principio 4: Robusto
El contendio de nuestra página web debe ser fiable esto significa que conforme la tecnología avance el contenido de la página debe de actualizarse
  1. Compatible La página debe de maximizar la compatibilidad con las aplicaciones actuales y futuras.

Normativa y sanciones

¿Cúales son los requisitos legales que se deben cumplir para la creación de una página web?

Para tener una pagina web en internet debes tener en cuenta la existencia la de la ley 34/2002, de 11 de julio, de servicios de la sociedad de la información y de comercio electrónico. la llamada "ley de internet" o LSSI.

Aún asi debe tener en cuenta que esta ley no siempre le será de aplicación ya que esto dependerá de si usted percibe o no compensación económica por su página.

¿Cuándo podemos decir que una página web tiene actividad económica?

Se considerará que esta página recibe compensación económica cuando perciba ingresos directos (Comercio) o indirectos (publicidad).

Por tanto no se le aplicara LSSI cuando su pagina web no realize ningún tipo de actividad económica. En el caso de que su página tenga publicidad pero no por ello obtenga becenificio económico queda exento de esta ley. En caso contrario automáticamente se convertirá en prestador de servicio de la Sociedad de la información y deberá analizar la aplicación que le corresponda según la ley.

¿Qué es un prestador de servicio de la Sociedad de la información?

Cualquier persona física o jurídica que proporciona un servicio prestado normalmente a título oneroso, a distancia, por vía electrónica y a petición individual del destinatario, será considerado a efectos de la Ley como Prestador de Servicios de la Sociedad de la Información.

Será por tanto servicios de la Sociedad de la Información las siguientes actividades económicas:

  1. La contratación de cualquien bien por vía electrónica
  2. La organización de subastas o de comercios virtuales
  3. La gestión de compras en la red por grupos de personas
  4. El envío de comunicaciones comerciales
  5. El suministro de información
Requisitos legales que impone LSSI
A-Obligación de información general
Tenemos la Obligación de información general recogidas en el articulo 10 citado anteriormente, asi pues y según dicta la ley debes indicar lo siguiente:
  1. Nombre o denominación social y datos de contacto
  2. Datos de inscripción en el Resgistro Mercantil
  3. NIF
  4. Información referente al precio del producto que se va a verder además de gastos de envio y cualquier tido de impuesto sujetos a él
  5. En el caso en que la actividad económica que se ejerza precise de una autorización administrativa previa, deberan acreditarse dichos datos
  6. Si se ejerce un prefesión regulada deberá acreditarse el titulo académico.
  7. Los códigos de conducata a los que esté adherido, en su caso, y la forma de consultarlos electrónicamente
  8. Información acerca del uso de cookies. No notificar esto será considerado una falta grave ta y como se estipula en el artículo 38
B-Obligación relativa a la contratación
  1. Antes de iniciar el procedimiento de contratación, deberá poner a disposición del usuario, mediante técnicas adecuadas al medio de comunicación utilizado, de forma permanente, fácil y gratuita, información clara, comprensible e inequívoca sobre:
    • Los trámites que deben seguir para realizar el contrato
    • Si se va a archivar el documento electrónico y si sera visible
    • Los medios técnicos que pone a su disposición para identificar y corregir los errores en la intrucción de los datos antes de confirmarlos
    • La lengua en la que se formaliza el contrato
    • Las condiciones que regirán el contrato
  2. Celebrado el contrato, el prestador debe confirmar la recepción de la aceptación, ya se por medio de correo electrónico u otro medio de comunicación equivalente.

    Las anteriores oblicaciones quedan excluidas solo en dos casos: si hay acuerdo por ambas partes y nunguna de ellas tuviera las condiciones de consumidor o cuando el contrato se ha llevado acabo en su totalidad por correo electrónico o por una vía equivalente

Sanciones

Pueden distinguirse tres tipos de faltas a la hora de cometer errores en la accesibilidad de una página web:

  1. Leves. Se consideran faltas leves aquellas irregularidades formales en lo establecido en la Ley 51/2003, del 2 de diciembre (igualdad de oportunidades, no discriminación y accesibilidad universal de las personas con discapacidad). Esta ley es complementaria y ampliada por el Real Decreto 1494/2007, del 12 de noviembre, por el que se aprueba el Reglamento sobre las condiciones básicas para el acceso de las personas discapacitadas a las tecnologías, productos y servicios TIC, además de los medios de comunicación social.

  2. Graves. Se consideran como infracciones graves:

    1. El incumplimiento de las exigencias de accesibilidad, además de no adoptar las medidas de ajuste razonable, que se definen en el artículo 7.b y c de la Ley 51/2003, del de 2 de diciembre, de igualdad de oportunidades, no discriminación y accesibilidad universal de las personas con discapacidad y en sus normas de desarrollo.

    2. El incumplimiento de un requrimiento administrativo que formulen los órganos competentes para el ejercicio de las competencias necesarias para dar cumplimiento a las previsiones de la Ley 51/2003, de 2 de diciembre, de igualdad de oportunidades, no discriminación y accesibilidad universal de las personas con discapacidad.

  3. Muy graves. Se considera como falta grave el incumplimiento reiterado de los requerimientos administrativos específicos que formulen los órganos competentes para el ejercicio de las competencias necesarias para dar cumplimiento a las previsiones de la Ley 51/2003, de 2 de diciembre, de igualdad de oportunidades, no discriminación y accesibilidad universal de las personas con discapacidad y en sus normas de desarrollo.

Asimismo, las sanciones varían dependiendo del tipo de falta:

Mínimo Medio Máximo
Leves 301 – 6.000 6.001 – 18.000 18.001 – 30.000
Graves 30.000 – 60.000 60.001 – 78.000 78.001 – 90.000
Muy graves 90.001 – 300.00 300.001 – 600.000 600.001 – 1.000.000

Cómo enlazar CSS

Para enlazar un archivo externo CSS para nuestro HTML, se utiliza la etiqueta <link>

<link rel="stylesheet" type="text/css" href="/css/estilos.css" />

Donde en los atributos rel y type se especifica que es una hoja de estilos y href indica el directorio donde se encuentra nuestro CSS.

Colores y fondos en CSS

Colores

Para modificar los colores de cualquier parte de la página basta con escribir dentro de un archivo CSS el nombre de la etiqueta a modificar seguido de dos llaves que encierran la modificación que queremos realizar. Cada modificación siempre debe terminar en un punto y coma. Por ejemplo, para modificar el color de texto y del fondo del cuerpo:

body {

color: green;

background-color: red;

}

Los colores pueden definirse de las siguientes formas:

  1. Con una combinación de rojo, verde y azul:

    color: rgb(cantidadDeRojo, cantidadDeVerde, cantidadDeAzul);

    El máximo para cada color es 255. Puedes ayudarte de algún programa como Visual Studio Code para dar con la combinación perfecta de estos tres colores para tu página.

  2. Con un color en hexadecimal:

    color: #884dff;

    Puedes utilizar el selector de color de W3Schools para encontrar el código de tu color ideal.

  3. Simplemente escribiendo el nombre del color en inglés:

    color: blueviolet;

    Aquí tienes una lista de nombres de colores que puedes utilizar.

Fondos

Se añaden escribiendo background- delante del nombre del tipo de fondo que vayamos a utilizar.

  1. Imagen de fondo. Se consigue escribiendo background-image: url(directorio de nuestra imagen);
  2. Color de fondo. Como ya se ha explicado en la sección Colores, se escribe background-color: tucolor ;
  3. Colocación de la imagen. Para ello se utiliza background-repeat, background-position, background-attachment

Otras propiedades

"font-variant :normal|small-caps;" para especificar texto con small-caps o sin el (letras de tamaño menos al original). "line-height: distancia;" para calibrar el interlineado o distancia entre líneas se puede indicar con un número, por ej 2 sería el doble o un número seguido de una unidad de medida como px. @font-face{font-family: fuente; src:url ('fuente.ttf'); } para precargar la letra necesaria

Word Spacing

Indica el espacio que hay entre las palabras de un texto, ej: word-spacing: 10px;

Se utiliza para transformar el texto a mayúsculas o minúsculas, ej: text-transform: uppercase;, los demás valores posibles serían capitalize (para la primera letra en mayúscula), lowercase (minúscula) y none (sin modificación).

Otras Propiedades

Otras propiedades mencionables serían "direction: dirección;", para establecer la dirección hacia la que se escribe el texto, ltr para izquierda a derecha y rtl para derecha a izquierda. "text-overflow = opcion;", se utiliza para que cuando un texto no quepa en un div, este se recorte si se usa la opción clip o se recorte y se añadan ... al final si se usa la opción ellipsis. Y por último "text-shadow: color distanciaX distanciaY desenfoque;", que da un efecto de sombra al texto, los valores distanciaX (desplazamiento horizontal), distanciaY (desplazamiento vertical) y desenfoque son numéricos y los dos primeros pueden ser negativos

Clases e IDs

Las clases

Una clase es una manera de agrupar elementos CSS del mismo tipo, de manera que se puedan utilizar como un atributo en la etiqueta en la que se desee. Tiene la misma forma que una selección estándar de elementos pero se comienza con ".NOMBRE DE CLASE" y sólo puede definirse en la etiqueta <style>. Si se quiere que la clase sólo pueda usarse con un elemento concreto, se escribe "NOMBRE DEL ELEMENTO.NOMBRE DE CLASE"
